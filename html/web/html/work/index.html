br 태그

요소는 줄바꿈을 나타냅니다. 이 요소는 줄바꿈을 표현하는 용도가 아니라 주소 또는 시처럼 줄바꿈이 실제 내용의 일부분일 때 씁니다.

img 태그

태그는 이미지를 삽입하는 태그로, scr 속성을 통해 이미지 경로를 지정합니다. 이미지 파일이 scr 속성에서 지정한 경로에 없을 시, 이미지는 출력되지 않거나 엑스박스가 뜨게 됩니다.
  • scr : source의 약자로, href 속성처럼 주소를 입력합니다.
  • border : 이미지 테두리 폭을 지정 (pixel값)
  • width : 사진 너비를 지정(pixel값)
  • height : 사진 높이를 지정(pixel값)
  • align : 사진 정렬
  • alt : 사진이 뜨지 않을 때 이를 대체할만한 설명

table 태그

데이터를 담고있는 표를 만드는 태그
  • colspan : column span 약자로, 셀(가로줄)을 합치는 개수를 지정
  • sowspan : row span이란 뜻으로 셀(세로줄)을 합치는 개수를 지정
  • border : 테이블 경계선 굵기를 지정(예: border="10")
  • width : 너비를 지정(픽셀이나, %)
  • height : 높이를 지정(픽셀이나, %)
  • cellpadding : 셀과 경계선 사이의 여백(그림 A는 cellpadding="10"으로 지정한 화면)
  • cellspacing : 셀과 셀 사이의 여백(그림 B는 cellspacing="10"으로 지정한 화면)
  • align : 셀의 가로줄을 오른쪽, 왼쪽, 중앙등으로 정렬
  • valign : 셀의 세로줄의 위, 중앙, 아래를 정렬
  • background-color : 배경색 지정
  • border-color : 경계선 색깔 지정

form 지정

입력된 데이터를 한 번에 서버로 전송합니다. 전송한 데이터는 웹 서버가 처리하고, 결과에 따른 또 다른 웹 페이지를 보여줍니다.
  • action : 양식이 전송될 때 양식 데이터를 전송할 주소
  • method : 양식 데이터를 전송할 때 사용할 HTTP 방식
  • target : 양식이 어느 창에 열릴지 지정
  • accept : 쉼표로 분리된 내용 유형 목록(html5에서 퇴화 속성)
  • accept-charset : 양식 전송에 사용될 문자 부호화의 목록
  • enctype : 서버에 전송할 때(method="post"일 때)내용 유형을 지정
  • autocomplete : 양식의 자동완성 지정
  • novalidate : 전송될 때 양식은 유효화하지 않아야 함을 지정

입력양식 - form

사용자가 입력한 정보를 서버로 전송할 때 사용하는 태그입니다.
  • 전체를 form태그로 감싸줍니다.
  • action="전송받는 서버url" 속성 : 전송받는 서버의 주소를 입력합니다.
  • <input> : 사용자로부터 정보를 입력받을 수 있는 태그입니다.
  • type="text" 속성 : 사용자로부터 문자 정보를 받을 수 있는 박스를 만듭니다.
  • type="password" 속성 : password에 해당되는 정보들이 **으로 표시됩니다.
  • type="submit" 속성 : 사용자가 입력한 정보가 서버로 전송되는 버튼을 만듭니다.
  • name="id" , name="pwd", name="address" 속성을 사용, 각각의 박스에 이름을 정해줍니다.
  • *name이라는 속성은 모든 폼태그에서 필요한 속성입니다.*
  • 각각의 이름의 값으로 서버에 데이터가 전송됩니다.

form - 텍스트 입력

사용자로부터 텍스트를 입력받는 태그입니다.
  • value="default value" 속성 : 사용자가 페이지를 열었을 때 기본적으로 적혀있는 텍스트를 설정합니다.
  • <textarea> : 여러 줄을 입력 받을 때는 textarea태그를 사용해야 합니다.
    속성을 이용하여 textarea를 설정해줍니다.
    cols, rows 속성을 사용하여 입력칸의 박스 크기를 변경할 수 있습니다.

form - 선택(dropdown list, combo box)

제한된 공간안에서 여러 개의 선택지를 선택할 수 있도록 하는 태그입니다.
  • 선택지들은 <option>로 감싸줍니다.
  • 선택을 할 수 있게 <option>로 감싼 선택지들을 <select>로 묶어줍니다.
  • <select name=""> name을 지정하여 서버에 전송 될 수 있게 적어줍니다.
  • <option value=""> value값을 지정하여 서버에 전송 될 수 있게 적어줍니다.
  • 다중선택 : <select>에 속성으로 multiple이라는 값을 넣어줍니다.
    값은 따로 없습니다. ctlr을 눌러야 복수선택이 가능합니다.

form - radio, checkbox

사용자가 여러개의 선택지 중에 하나 또는 복수개를 선택해서 제출할 수 있도록 하는 태그입니다.
  • radio 버튼 (단일선택)
    • <input type="radio"> 동그란 체크박스을 설정합니다.
    • 여러개를 만들었을 때는 name을 똑같이 지정해주면 같은 이름으로 지정되어 있는 것들끼리는 하나의 버튼만 선택됩니다.
    • 라디오 버튼은 네임값이 같은 것들 끼리 그룹핑이 되는 것이고
      같은 그룹에 속하는 것 중에서 하나가 선택되면 나머지의 선택이 해제되는 형태의 버튼입니다.
  • checkbox(다중선택)
    • 라디오 버튼과는 다르게 같은 name을 가지고 있는 여러개의 선택지들을 다중으로 선택할 수 있습니다.
    • 여러개를 만들어서 name을 똑같이 지정해주면 같은 이름으로 지정되어 있는 것들끼리 그룹핑 해줍니다.
    • 사용자가 웹페이지를 열었을 때 기본적으로 어떠한 항목이 선택되어 있게 하고 싶다면
      'checked'라고 하는 값이 없는 속성을 넣어줍니다.

form - 버튼

  • <input type="submit" value="전송"> : 값이 전송되는 서버로 페이지가 이동됩니다.
  • <input type="button" value="버튼" : 아무런 동작도 하지 않습니다.
  • onclick="alert('hello world')"> 자바스크립트를 사용할 때 활용도가 생기는 속성입니다.
  • <input type="reset"> : 재설정, 사용자가 입력했던 모든 정보가 초기화하는 버튼입니다.

form - 컨트롤의 제목:label

  • html에서 권장하는 방법입니다.
  • <label>#</label> : '#'은 단순히 텍스트가 아니라 '무언가의 이름표'라는 의미를 갖게됩니다.
  • label이 누구의 이름표인지를 표현해주기 위해서 'for'속성을 사용해야 합니다.
  • label 태그의 for속성의 이름과 = input 태그의 'id'속성의 이름이 같아야 합니다.
  • for속성과 id속성을 따로 입력해주지 않는다면 label태그로 form태그까지 감싸주면 됩니다.
    표현방법이 다를 뿐 같은 결과로 나옵니다.
  • input type="checkbox" 형식도 label태그로 전체를 감싸면 체크박스를 클릭하거나 텍스트를 클릭해도 선택이 됩니다.

form - method

사용자가 입력한 데이터를 서버에 어떠한 방식으로 전송할 것인가 전송 방법의 타입을 지정합니다.
  • form태그에 속성으로 method="get or post"방식을 지정해줍니다.
  • get방식 : 기본값 , url을 통해서 데이터를 전송하는 방식
  • post방식 : url이 아니라 다른 방법으로 데이터를 숨겨서 전송하는 방식, 100% post값으로 전송하면 됩니다.
  • 방식의 결정권은 서버 엔지니어에게 있습니다.
  • 데이터를 전송시 기본적으로 method="post" 방식으로 지정해서 처리하면 됩니다.

form - 파일 업로드

  • <input type ="file"> : 사용자가 파일선택을 누르면 파일을 자기 컴퓨터에서 선택할 수 있는 화면이 뜹니다.
  • <form method="post" enctype="multipart/form-data>을 속성을 처리해주어야 합니다.

form - 데이터 전송:hidden field

ui가 없지만 몰래 서버로 어떤 값을 전송하고 싶을 때 사용하는 태그입니다.

정보로서의 HTML

  • HTML이 처음 고안되었을 때는 단지 웹페이지를 표현하기 위한 언어였음.
  • 웹이 폭발적으로 성장하면서 인류의 많은 정보들이 '웹 화' 됨!
  • 인류의 지식을 담아내는 그릇같은 역할은 합니다.
  • 정보를 표현하고 정보를 담아내는 역할에 집중하고 전념할 수 있게 하기 위한 여러가지 방법들이 고안됩니다.

글꼴 - font (퇴출됨)

  • 대표적으로 퇴출된 태그 중 하나입니다.
  • size="" 속성을 사용하여 폰트의 크기를 설정할 수 있습니다. 1-7까지
  • color=""속성을 사용하여 폰트의 색상을 설정할 수 있습니다.
  • font태그는 시각적인 디자인일 뿐이지 어떠한 정보도 가지고 있지 않습니다.
  • HTML파일의 용량도 커지고 디자인과 관련된 코드와 정보와 관련된 코드가 뒤섞여서
    웹페이지의 정보로서의 가치가 현격히 떨어집니다.
  • 디자인과 관련된 폰트 태그와 같은 종류의 태그들을 퇴출시키면서 'CSS'라고 하는 별도의 프로그래밍 언어를 고안합니다.

<meta>

데이터를 설명하는 데이터를 meta data라고 합니다.
  • 글의 본문에 표현되지는 않지만, 웹페이지 또는 글의 본문을 설명하는 중요한 정보들입니다.
  • charest(문자)="uft-8"속성을 사용하여 저장된 방식을 표현합니다.
  • name="description" content="소개자료" : 요약자료
  • name="keywords" content="코딩,생활코딩" : 문서를 정의하는 중요한 단어들
  • name="author" content="이름" : 저자
  • http-equiv="refresh" content="30" 30초 간격으로 자동으로 refresh가 됩니다.
  • meta태그를 잘 작성하면 웹페이지가 훨씬 더 가치있는 정보가 될 수 있습니다.
